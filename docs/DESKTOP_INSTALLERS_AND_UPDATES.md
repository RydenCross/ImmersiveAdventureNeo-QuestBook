# Windows Installer, Linux AppImage, and Update Metadata

FTB Quest Maker can turn the target-specific PyInstaller binary into a normal desktop package and generate deterministic update metadata for distributing those packages.

## Package build plans

Inspect the Windows installer plan without invoking Inno Setup:

```bash
python -m generator quest-maker-package-build \
  --platform windows \
  --version 1.0.0 \
  --dry-run
```

Inspect the Linux AppImage plan:

```bash
python -m generator quest-maker-package-build \
  --platform linux \
  --version 1.0.0 \
  --dry-run
```

A real build expects the native binaries generated by `quest-maker-native-build`:

```text
dist/native/FTBQuestMaker.exe
dist/native/FTBQuestMaker
```

Windows packaging uses Inno Setup and emits:

```text
dist/packages/FTBQuestMaker-1.0.0-Setup.exe
```

Linux packaging stages an AppDir with `AppRun`, desktop metadata, an SVG icon, and the executable before invoking `appimagetool`:

```text
dist/packages/FTBQuestMaker-1.0.0-x86_64.AppImage
```

Package builds must run on their target operating system. Dry runs are platform-independent and validate the deterministic plan and checked-in recipes.

Helper recipes are available at:

```text
packaging/windows/ftb_quest_maker.iss
packaging/windows/build_installer.ps1
packaging/linux/AppRun
packaging/linux/build_appimage.sh
packaging/linux/ftb-quest-maker.svg
```

## Update metadata

Generate update metadata from one or both built packages:

```bash
python -m generator quest-maker-update-metadata \
  --version 1.0.0 \
  --channel stable \
  --artifact windows=dist/packages/FTBQuestMaker-1.0.0-Setup.exe \
  --artifact linux=dist/packages/FTBQuestMaker-1.0.0-x86_64.AppImage \
  --base-url https://downloads.example.com/ftb-quest-maker/1.0.0 \
  --destination dist/updates/latest.json
```

Supported channels are `stable`, `beta`, and `nightly`. Stable metadata rejects prerelease semantic versions.

Each artifact record contains its platform, filename, download URL, byte size, and SHA-256 digest. Records are sorted by platform, and the JSON encoding is deterministic.

### Authenticated metadata

An optional secret key can authenticate the canonical metadata payload with HMAC-SHA256:

```bash
python -m generator quest-maker-update-metadata \
  --version 1.0.0 \
  --channel stable \
  --artifact windows=dist/packages/FTBQuestMaker-1.0.0-Setup.exe \
  --signing-key /secure/release-update.key \
  --key-id release-2026 \
  --destination dist/updates/latest.json
```

Signing keys are local release secrets. `.key` and `.pem` files are ignored by repository hygiene and must never be committed or bundled with the application.

Verify metadata, its signature, and local package bytes:

```bash
python -m generator quest-maker-update-verify \
  dist/updates/latest.json \
  --artifact-directory dist/packages \
  --signing-key /secure/release-update.key
```

Verification rejects malformed schemas, duplicate platforms, unsafe filenames, invalid sizes or hashes, missing artifacts, changed artifact bytes, missing signing keys, and signature mismatches.

## Audit

Run the installer and update contract:

```bash
python -m generator desktop-packages-audit
```

Generate the tracked report:

```bash
python -m generator desktop-packages-audit \
  --format json \
  --output reports/desktop-packages-audit.json
```

The audit verifies deterministic Windows and Linux package plans, platform-specific artifact naming, target-only builds, Linux AppDir staging, deterministic signed update metadata, correct-key verification, wrong-key rejection, tamper detection, and the checked-in package recipes.
